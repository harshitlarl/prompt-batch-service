Every Endpoint, One Page

App API, dashboard pages, and admin/ops endpoints — what they do and whether they're reachable from here.

App API · port 8080 · public

GET / Redirects (303) to /ui
GET /ping Liveness check
POST /batches Submit a batch — body {"prompts": ["..."]} → try it on Submit
POST /batches/upload Submit a batch via file upload (.txt one-per-line or .json array)
GET /batches List every batch, most recent first → see it live on Task Status
GET /batches/{id} One batch's live progress
GET /batches/{id}/results Per-prompt results (409 until finished)

Dashboard & API Docs · port 8080 · public

GET /ui Submit page — paste/upload prompts, submit a batch, watch it live
GET /ui/status.html Task Status page — all batches, live-polling progress table
GET /swagger Interactive Swagger UI
GET /openapi.json OpenAPI 3 spec (JSON)
GET /openapi.yaml OpenAPI 3 spec (YAML)

Admin / Ops · port 8081 · internal only

GET/healthcheckJSON health of every registered check (ping, deadlocks, worker pools, database)
GET/metricsFull Dropwizard/JVM metrics dump
GET/threadsLive JVM thread dump
GET/pingAdmin connector's own liveness check
GET/pprof/profileOn-demand CPU profile capture
GET/pprof/contentionOn-demand lock-contention profile capture
POST/tasks/log-levelChange a logger's level at runtime
POST/tasks/gcTrigger a JVM garbage collection
Full write-up with request/response shapes: docs/API_ENDPOINTS.md in the repo.
